# 2. Реалізація ООП парадигм

У файлі `agent.py` реалізовано систему прогнозування погоди з демонстрацією чотирьох основних принципів об'єктно-орієнтованого програмування.

## 2.1 Абстракція

Створено абстрактний клас `Sensor`, який визначає загальний інтерфейс для всіх датчиків.

```python
class Sensor(ABC):
    @abstractmethod
    def read(self):
        pass
```

Абстрактний метод `read()` зобов'язує дочірні класи реалізувати власний механізм отримання показників.

---

## 2.2 Наслідування

Від класу `Sensor` успадковано два спеціалізовані класи:

* `TemperatureSensor`
* `HumiditySensor`

```python
class TemperatureSensor(Sensor):
    ...

class HumiditySensor(Sensor):
    ...
```

Ці класи використовують спільний інтерфейс базового класу та реалізують власну логіку роботи.

---

## 2.3 Інкапсуляція

У класі `WeatherStation` список датчиків зберігається у приватному атрибуті:

```python
self.__sensors = []
```

Доступ до нього здійснюється лише через публічний метод:

```python
add_sensor()
```

що забезпечує контроль над внутрішнім станом об'єкта.

---

## 2.4 Поліморфізм

Під час формування звіту метеостанція викликає метод `read()` для кожного датчика без знання його конкретного типу.

```python
for sensor in self.__sensors:
    value = sensor.read()
```

Залежно від об'єкта виконується відповідна реалізація методу.

# 5. Теоретичні питання

## Що таке абстрактний клас?

Абстрактний клас — це клас, який не можна створити напряму. Він використовується як шаблон для інших класів та може містити абстрактні методи, що обов'язково реалізуються в дочірніх класах.

---

## Що таке наслідування?

Наслідування дозволяє створювати нові класи на основі існуючих та повторно використовувати їх функціональність.

---

## Що таке інкапсуляція?

Інкапсуляція забезпечує приховування внутрішніх даних об'єкта та доступ до них через спеціальні методи.

---

## Що таке поліморфізм?

Поліморфізм дозволяє використовувати однаковий інтерфейс для роботи з різними типами об'єктів.

---

## Для чого використовується модуль random?

Модуль `random` використовується для генерації випадкових значень температури та вологості, що імітує роботу реальних датчиків.
